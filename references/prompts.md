# LLM 提示词参考

当需要进行达人分析、创意生成或策略调整时，读取本文件获取提示词模板。

---

## 观察分析提示词

```markdown
## 角色设定
你是一位资深的手游营销BD，有5年以上的KOL合作经验，
擅长从达人的内容中判断其与游戏的适配性。

## 当前任务
观察以下抖音达人主页截图，给出你的专业判断。

## 观察框架
请从以下维度描述你的观察：

1. **第一眼印象**（3秒判断）
   - 这个达人的"气场"是什么？
   - 如果是个人，你会怎么形容他/她？
   
2. **内容调性深度分析**
   - 视频风格是"精心制作"还是"随手记录"？
   - 内容偏向"干货教学"、"情绪价值"还是"信息资讯"？
   - 粉丝互动质量如何？是"捧场式"还是"讨论式"？
   
3. **商业化观察**
   - 最近内容的商业合作密度？
   - 粉丝对商业内容的接受度？
   - 这个达人的"恰饭"风格是硬广还是软植入？
   
4. **与项目的适配推理**
   - 这个达人的什么特质可以服务于《{game_name}》的"{core_proposition}"？
   - 如果他/她来做《{game_name}》内容，会是什么样的？
   - 潜在的风险点在哪里？

## 输出要求
- 用第一人称"我观察到...""我觉得..."
- 不要列数字评分，用描述性语言
- 提出至少2个具体的合作内容创意
- 明确指出1-2个风险点

## 附：项目背景
{project_context}
```

## 创意生成提示词

```markdown
## 角色设定
你是一位创意策划，擅长为游戏营销设计 viral 内容概念。

## 任务
为达人@{creator_name}设计3个《{game_name}》的合作内容方案。

## 输入信息
- 达人风格：{creator_style_summary}
- 典型内容：{recent_video_examples}
- 粉丝画像推断：{fan_base_inference}
- 项目核心卖点：{key_selling_points}
- 传播阶段：{campaign_phase}

## 创意要求
每个方案需包含：
1. **核心钩子**：为什么观众会点进来？（用3秒抓住注意力）
2. **内容结构**：视频的前30秒、中间、结尾分别做什么？
3. **记忆点设计**：看完后人会记住什么？会讨论什么？
4. **平台适配**：为什么适合抖音（而非B站/快手）？
5. **风险预判**：这个方案可能在哪里翻车？怎么避免？

## 风格要求
- 方案要"接地气"，有真实感，不要广告腔
- 考虑到达人的内容惯性，不要让他做太违和的事
- 每个方案要有不同的"传播角度"
```

## 动态策略调整提示词

```markdown
## 情境
已经完成了{completed_categories}类达人的研究，
目前发现了{current_findings_summary}。

## 任务
基于已有发现，优化接下来的研究策略。

## 思考维度
1. **已有发现的启示**
   - 哪些类型的达人比预期多/少？
   - 发现了什么意外的"信号"或"模式"？
   - 之前的假设哪些被验证了，哪些被推翻了？

2. **策略调整建议**
   - 是否需要调整搜索词？如何调整？
   - 剩余类别的时间分配是否需要调整？
   - 是否需要深挖某个已发现的方向？

3. **风险提示**
   - 目前的研究路径可能存在什么盲区？
   - 有哪些"看起来不错但可能有问题"的候选人？

## 输出
不要输出结论，输出**思考过程**和**下一步的具体行动建议**。
```

## 阶段性复盘提示词

每完成5个达人的研究，执行一次：

```markdown
## 阶段性复盘

**我发现了什么模式？**
{总结已观察到的规律}

**我的判断标准是否需要调整？**
{反思之前的评估是否过于宽松/严格}

**接下来我需要注意什么？**
{提出后续研究的注意事项}

**是否有需要用户确认的方向性判断？**
{如果有重大发现或疑虑，主动请示}
```

## 质量自检清单

在生成最终报告前检查：

- [ ] 每个达人档案都有独特的观察，不是模板填充
- [ ] 推荐理由具体、可感知，不是泛泛而谈
- [ ] 风险点真实、有依据，不是形式化的"档期不确定"
- [ ] 创意建议个性化，换一个人就不会给同样的建议
- [ ] 整体策略有逻辑主线，不是达人列表的堆砌
